Managing User Permissions

Project Creation and Management ››


A primary concern of researchers and data mangers is data control. VOEIS provides highly granular user access controls that allow PIs to grant varying degrees of read and/or write permissions to any registered VOEIS user. After you create a new project in VOEIS, you are able to select who may use your project and grant various levels of read/write priveleges each individual.  




       

The access granted to individuals occupying each role are listed below.                

       

User Type

Role Description

Principal Investigator

Principal Investigators create projects to pursue research goals. Can add data and browse data, manage CV, modify users in the project, publish data.

Program Manager

Program Managers for the research project. NSF program manager only - browse data

Data Manager        

Data Managers manage all the data for a project. Can add data and browse data, manage CV, modify users in the project and publish data.

Field Technician

Field Technicians manage field activities. Can add data and browse data

Laboratory Technician

Lab Technicians manage lab activities. Can add data and browse data.

Member

General members of projects. Can add data and browse data

Observer

Can browse project data only.        




After you add users to your project and define their respective roles, you will see a list at the bottom of the window as shown below. Preview the list to ensure that you assigned the correct roles to the correct individuals.







It is possible to edit user permissions at any time within a project. You can remove a user from your project by clicking the "Delete" button in the row corresponding to the individual's name. It is also possible to adjust individual user's roles within the project. Click the "Edit" button to open a dialog box like the one displayed below:






Use this dialog box to adjust the user's settings within the project and click the "Update" button to save your changes.
