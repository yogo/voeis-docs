Loading Data


This section describes the use of the wizards provided by the VOEIS Data Hub. The data upload wizards greatly simplify the task of data archiving in VOEIS. The wizards allow users to create data upload parsing templates that can be saved and used repeatedly for future uploads of data files that exhibit identical formatting. Template creation is simple and intuitive and users can preview thier parsing instructions prior to uploading data to VOEIS.


Understanding the Concept of the Variable


Before endeavoring to manage variables in a Project, it is important that to understand the concept of the variable in VOEIS. A variable in VOEIS is more than just a variable name like 'Temperature'. A variable is an entire collection of information (or metadata) about an individual or multiple data values that includes the following:


  • Variable Name
  • Units
  • General Category
  • Speciation
  • Data Type
  • Sample Medium
  • Quality Control Level
  • Spatial Offset Type
  • Spatial Offset Value
  • Spatial Offset Units
  • Time Collection Interval
  • Collection Interval Units
  • Field Collection Method
  • No Data Value
  • Sensor Type
  • Logger Type


These fields are modeled after the conceptualization of the variable in the Observation Data Model (ODM) used by the CUAHSI-HIS. The terms associated with each of these fields and avaialble for use in the creation of variables in any VOEIS project is discussed in detail here. More information about each of the above fields may be found in the ODM documentation here. It is strongly recommended that users review this documentation. It is critical that users understand the concept of the variable when they endeavor to save data to VOEIS using the data upload wizards because this is the basic metadata set used to describe data values. Incorrect assignment of variables to data values will significantly reduce the quality and usability of data stored in a VOEIS project.
